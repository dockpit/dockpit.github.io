dockpit

turbo-charge your microservice development using docker

twitter github

Quickly describe the API and dependencies of you service just by writing simple black-box test cases.

Develop a single service in isolation by mocking all dependencies and set up the addresses as environment variables with a single command. 

$> pit mock
 - users [1.1.0] mocked @ 192.168.1.10
 - orders [0.2.0] mocked @ 192.168.1.11

When dependencies are mocked you can run the test cases quickly and reliably to make sure the service is implemented correctly.

$> pit test
 - address_list/”return 200” - OK
 - address_create/”redirect to index”- OK
 - address_create/”fail on duplicate” - OK
   All passed took 150ms

Semantic versioning of you service can be done automatically. Minor version numbers are incremented whenever breaking changes in the API occurs.

$> pit changelog
22-09-2014 0.1.11 -> 0.2.0
 -breaking change in “list_addresses'

Get informed immediately whenever a service you depend on had a breaking change and your implementation needs to be updated.

$> pit test
warning! service ‘users’ updated to ‘0.2.0’
 - review address_list/”return per user”

Up-to-date API documentation by generating API blueprint files with a single command

$> pit blueprint
# GET /message
+ Response 200 (text/plain)

   Hello World!
